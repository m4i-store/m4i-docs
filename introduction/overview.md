# Overview

`m4i-docs` organizes M4I knowledge in four sections:

- Introduction: onboarding and docs model
- Core Systems: platform-level systems (for example `m4i_bridge`)
- Scripts: script-level documentation and templates
- Shared Standards: cross-project rules and policies

## Documentation model

- Central hub first: full docs live here
- Script repos second: short README + link to official docs page
- One change path: documentation updates via this repository

## Scale strategy

As M4I grows, this structure avoids documentation sprawl by keeping:

- one canonical location
- one navigation tree
- one standards layer reused by all scripts
